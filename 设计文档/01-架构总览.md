# ToolBox 架构总览

> 版本：v1.0
> 文档定位：项目顶层设计，描述整体架构、技术选型、模块划分、核心流程、配置与部署。后续的插件开发指南、组件 API 参考、部署运维指南等文档基于本文展开。

---

## 1. 项目背景与目标

### 1.1 背景

团队在日常工作中需要一系列内部小工具：执行预设 SQL 查询并加工结果、批量处理 Excel、把数据库里的关联关系渲染成流程图、跑各类一次性脚本等。这些工具零散、各人各写、难以复用和共享。

为了解决这个问题，本项目搭建一个**统一的工具集容器**：

- 提供一个壳，承载各种业务工具
- 工具以**插件**的方式接入，新增工具不修改主程序代码
- 内置数据源管理、后台任务、文件上传下载、通用 UI 组件等能力，插件按需使用
- 部署在内网服务器上，团队通过浏览器访问
- 单人单机也能用（部署在自己电脑上即可）

### 1.2 目标

| 类别 | 目标 |
|---|---|
| 易扩展 | 插件 jar 丢进 `plugins/` 目录，重启后端即可生效，主程序无需修改 |
| 易部署 | 绿色版，自带 JRE，解压即用，可注册为 Windows 服务 |
| 易使用 | 浏览器访问，无需在客户端安装任何东西 |
| 多人协作 | 用户系统、共享数据源、各自独立的任务空间 |
| 跨平台开发 | Mac 上开发，打包出 Windows 部署包 |

### 1.3 非目标（明确不做）

- 不做生产级安全（不加密数据源密码、HTTP 而非 HTTPS）
- 不做国际化（中文写死）
- 不做插件热加载（重启生效）
- 不做插件间通信
- 不做任务断点续跑
- 不做 Web 端日志查看
- 不做立即踢下线（等 token 自然过期）

### 4.2 模块职责与依赖

```
                       ┌────────┐
                       │  api   │  ← 不依赖任何业务模块
                       └───┬────┘
                           │
              ┌────────────┴────────────┐
              ▼                         ▼
       ┌─────────────┐           ┌──────────────┐
       │  framework  │           │  插件 (jar)  │
       └──────┬──────┘           └──────────────┘
              │
              ▼
       ┌──────────┐
       │   app    │ ← 整合前端 + 后端，打绿色版包
       └──────────┘

       frontend ──→ frontend-sdk
       插件前端 ──→ frontend-sdk
```

**关键约束**：
- `api` 只声明接口和数据模型，不放任何实现
- 插件**只**依赖 `api` 和 `frontend-sdk`，不依赖 `framework` 内部类
- 这样保证插件不会意外耦合到框架的实现细节，未来重构 framework 不会破坏插件

## 5. 核心数据模型

### 5.1 用户（User）

```java
public class User {
    private String username;        // 唯一，作为 ID
    private String passwordHash;    // BCrypt
    private Role role;              // USER / ADMIN
    private boolean enabled;        // 是否启用
    private boolean mustChangePassword; // 首次登录或被重置后强制改密码
    private Instant createdAt;
    private Instant lastLoginAt;    // 可空
}

public enum Role { USER, ADMIN }
```

存储：`config/users.json`

### 5.2 数据源（DataSourceConfig）

```java
public class DataSourceConfig {
    private String id;              // UUID
    private String name;            // 显示名（唯一）
    private DbType type;            // MYSQL / OCEANBASE
    private String host;
    private int port;
    private String database;
    private String username;
    private String password;        // 明文（按需求不加密）
    private Map<String, String> params;  // JDBC 额外参数
    private PoolConfig poolConfig;  // 连接池配置（可空，用默认）
    private long version;           // 乐观锁
    private Instant createdAt;
    private Instant updatedAt;
}

public class PoolConfig {
    private int maximumPoolSize = 10;
    private int minimumIdle = 2;
    private long connectionTimeoutMs = 30_000;
    private long idleTimeoutMs = 600_000;
    private long maxLifetimeMs = 1_800_000;
}

public enum DbType { MYSQL, OCEANBASE }  // OB 用 MySQL 协议
```

存储：`config/datasources.json`

### 5.3 任务（Task）

```java
public class Task {
    private String id;              // UUID
    private String pluginId;        // 哪个插件提交的
    private String taskType;        // 插件自定义类型
    private String name;            // 显示名
    private String username;        // 任务归属
    private TaskStatus status;      // PENDING / RUNNING / SUCCESS / FAILED / CANCELLED
    private int progressPercent;    // 0~100
    private String progressMessage; // 当前阶段描述
    private Instant submittedAt;
    private Instant startedAt;
    private Instant finishedAt;
    private String errorMessage;    // 失败时的错误
    private List<TaskArtifact> artifacts; // 产出文件
}

public class TaskArtifact {
    private String name;            // 文件名
    private String path;            // 服务器路径
    private long sizeBytes;
    private String contentType;
}
```

存储：SQLite 数据库 `data/tasks.db`，每天定时清理 30 天前的任务及其产物

### 5.4 插件元信息（PluginDescriptor）

由 PF4J 从 `plugin.properties` 读取：

```properties
plugin.id=sql-runner
plugin.class=com.toolbox.plugin.sqlrunner.SqlRunnerPlugin
plugin.version=1.0.0
plugin.provider=ToolBox Team
plugin.description=执行 SQL 并查看结果
plugin.dependencies=
# 自定义字段
toolbox.category=数据查询
toolbox.icon=mdi-database
toolbox.frontendEntry=frontend/index.js
```

## 6. 关键流程

### 6.1 应用启动

```
1. JVM 启动 Spring Boot Application
2. 读取 application.yml (端口、JWT 密钥、SQLite 路径等)
3. 初始化 UserStore (读 config/users.json)
   · 如果 users.json 不存在或为空 → 自动创建 admin/admin123 (mustChangePassword=true)
4. 初始化 DataSourceStore (读 config/datasources.json，仅加载元数据)
   · 注意：连接池此时不创建，懒加载
5. 初始化 TaskStore (打开 SQLite，启动 30 天清理定时任务)
6. PluginManager 扫描 plugins/ 目录
   · 仅读取 plugin.properties，不实例化插件类（懒加载）
   · 把每个插件的 frontend/ 映射为 HTTP 路径 /plugins/{pluginId}/*
7. Spring Boot 监听 9090 端口，提供：
   · 静态资源 (前端 SPA)
   · /api/** REST 接口
   · /sse/** SSE 接口
   · /plugins/{pluginId}/** 插件前端资源
8. 启动完成
```

### 6.2 用户登录

```
浏览器                后端
  │                   │
  │ POST /api/login   │
  │ {username,        │
  │  password}        │
  │ ──────────────────>│
  │                   │ 1. 查 UserStore
  │                   │ 2. BCrypt 校验密码
  │                   │ 3. 生成 JWT (8h)
  │                   │ 4. 更新 lastLoginAt
  │                   │
  │ {token,           │
  │  mustChangePassword,│
  │  role}            │
  │ <──────────────────│
  │                   │
  │ 如果 mustChangePassword:
  │   跳转改密码页 (强制)
  │ 否则:
  │   跳主界面，token 存 localStorage
  │   后续所有请求带 Authorization: Bearer {token}
```

### 6.3 工具列表加载

```
1. 前端拿 token 请求 GET /api/plugins
2. 后端返回所有已注册插件的元信息列表（不加载插件代码）
   [{ id, name, description, category, icon, frontendEntry }, ...]
3. 前端按 category 分组渲染左侧工具树
```

### 6.4 用户点击工具（首次打开）

```
1. 用户在工具树上点击 "SQL 查询器"
2. 前端检查该插件的代码是否已加载到当前页面
   · 已加载 → 直接创建 Tab 实例
   · 未加载 → 动态 import('/plugins/sql-runner/index.js')
              加载 css /plugins/sql-runner/index.css
              缓存到内存（页面生命周期内不重复加载）
3. 调用插件 export 的 createView(toolboxApi)，得到 Vue 组件
4. 前端在 Tab 容器里挂载该组件，标题默认 "SQL 查询器"
5. 插件可通过 toolboxApi.setTabTitle(...) 改标题（例如选了数据源后变 "SQL 查询器 - DS1"）
```

### 6.5 数据源选择 + 立即建连

```
插件前端                      后端                       业务数据库
   │                          │                            │
   │ 用户选了 DS1              │                            │
   │ POST /api/datasources/   │                            │
   │   ds1/connect             │                            │
   │ ─────────────────────────>│                            │
   │                          │                            │
   │                          │ 检查 DS1 是否有连接池      │
   │                          │ 没有 → HikariCP 新建连接池 │
   │                          │ 尝试拿一个连接             │
   │                          │ ─────────────────────────> │
   │                          │ <───────────────────────── │
   │                          │ (拿到连接，立即归还)       │
   │                          │                            │
   │ {success: true,          │                            │
   │  connectionId}            │                            │
   │ <─────────────────────────│                            │
   │                          │                            │
   │ Tab 关闭时:               │                            │
   │ POST /api/datasources/   │                            │
   │   ds1/release             │                            │
   │ ─────────────────────────>│                            │
   │                          │                            │
   │                          │ 释放该 Tab 的占用引用       │
   │                          │ (连接池本身不销毁，下次还能用)│
```

**注意**：连接池是按数据源全局共享的，"释放"指的是把这次工具用过的连接还回池里，连接池本身仍然存活，等待其他用户/工具复用。

### 6.6 SQL 执行（同步查询）

```
插件前端                      后端                        数据库
   │                          │                            │
   │ POST /api/datasources/   │                            │
   │   ds1/query               │                            │
   │ {sql, params, timeout,   │                            │
   │  maxRows}                 │                            │
   │ ─────────────────────────>│                            │
   │                          │                            │
   │                          │ 1. 从连接池拿连接          │
   │                          │ 2. setQueryTimeout(timeout)│
   │                          │ 3. setMaxRows(maxRows)     │
   │                          │ 4. preparedStatement.execute│
   │                          │ ─────────────────────────> │
   │                          │ <───────────────────────── │
   │                          │ 5. 拼装 ResultSet           │
   │                          │ 6. 归还连接                 │
   │                          │                            │
   │ {columns, rows,          │                            │
   │  truncated}              │                            │
   │ <─────────────────────────│                            │
```

**安全控制**：
- `timeout` 默认 60s，可在请求里覆盖（不超过全局上限）
- `maxRows` 默认 100 万行，超出截断并标记 `truncated=true`
- SQL 安全（防止 DROP / DELETE 误操作）由插件自己负责，框架不拦截

### 6.7 后台任务执行

```
插件前端                       后端                          数据库
   │                          │                              │
   │ POST /api/tasks          │                              │
   │ {pluginId, taskType,     │                              │
   │  name, payload}          │                              │
   │ ─────────────────────────>│                              │
   │                          │ 1. 检查该用户并发数 <5       │
   │                          │ 2. 创建 Task (PENDING)       │
   │                          │ 3. 入队 ExecutorService      │
   │                          │ 4. 持久化到 SQLite           │
   │                          │                              │
   │ {taskId}                 │                              │
   │ <─────────────────────────│                              │
   │                          │                              │
   │ GET /sse/tasks/{taskId}  │                              │
   │ ─────────────────────────>│ (SSE 长连接)                 │
   │                          │                              │
   │                          │ 任务执行 (异步线程):         │
   │                          │   1. 调用插件的 TaskHandler  │
   │                          │   2. 插件通过 ProgressReporter│
   │                          │      报告进度 →               │
   │                          │      框架推送到 SSE          │
   │ event: progress          │                              │
   │ {percent, message}       │                              │
   │ <─────────────────────────│                              │
   │ ...                      │                              │
   │                          │                              │
   │ event: completed         │ 任务完成:                     │
   │ {artifacts}              │   状态置 SUCCESS / FAILED    │
   │ <─────────────────────────│   持久化到 SQLite            │
   │                          │   关闭 SSE 连接               │
```

**取消任务**：
```
POST /api/tasks/{taskId}/cancel
  ↓
调用 Future.cancel(true) → Thread.interrupt()
  ↓
插件代码需要响应 Thread.currentThread().isInterrupted()
（数据库查询的取消调用 Statement.cancel()）
```

**关闭 Tab 时的中断**：

## 7. 插件机制

### 7.1 插件包结构

```
sql-runner-1.0.0.jar
├── plugin.properties              ← PF4J 元信息 + 自定义字段
├── classes/                       ← 插件后端代码
│   └── com/toolbox/plugin/sqlrunner/
│       ├── SqlRunnerPlugin.class  ← 插件入口（继承 PF4J Plugin）
│       └── SqlRunnerExtension.class ← 实现框架扩展点
├── lib/                           ← 插件私有依赖（可选）
│   └── *.jar
└── frontend/                      ← 插件前端代码
    ├── index.js                   ← 入口文件 (ESM 格式)
    ├── index.css
    └── assets/                    ← 图片等
```

### 7.2 PF4J 集成要点

- 每个插件用独立的 `URLClassLoader`，避免依赖冲突
- 插件继承 `org.pf4j.Plugin`，有 `start()` / `stop()` 生命周期
- 插件通过 `@Extension` 注解声明扩展点
- 框架定义的核心扩展点：

```java
// 工具扩展点：每个工具实现这个接口
public interface ToolExtension extends ExtensionPoint {
    String getId();
    String getName();
    String getCategory();
    String getIcon();
    /** 提交后台任务时的处理器（可空） */
    TaskHandler getTaskHandler();
    /** 同步操作的处理器（可空） */
    SyncHandler getSyncHandler();
    /** 插件初始化（拿到 PluginContext） */
    void init(PluginContext context);
}
```

### 7.3 PluginContext

框架在插件启动时注入：

```java
public interface PluginContext {
    DataSourceManager getDataSourceManager();
    TaskManager getTaskManager();
    NotificationService getNotificationService();
    Logger getLogger();
    Path getDataDir();          // plugins-data/{pluginId}/
    String getPluginId();
    String getCurrentUser();    // 当前操作的用户名（线程上下文）
}
```

### 7.4 前端动态加载

主程序前端用动态 import：

```typescript
// frontend/src/plugin-loader/index.ts
async function loadPlugin(pluginId: string) {
  const cssUrl = `/plugins/${pluginId}/index.css`;
  const jsUrl = `/plugins/${pluginId}/index.js`;

  // 加载 CSS
  if (!document.querySelector(`link[data-plugin="${pluginId}"]`)) {
    const link = document.createElement('link');
    link.rel = 'stylesheet';
    link.href = cssUrl;
    link.dataset.plugin = pluginId;
    document.head.appendChild(link);
  }

  // 动态 import JS
  const module = await import(/* @vite-ignore */ jsUrl);
  return module.default; // 插件导出的工厂函数
}
```

插件前端入口：

```typescript
// 插件 frontend/src/index.ts
import { defineComponent } from 'vue';
import type { ToolboxAPI } from '@toolbox/frontend-sdk';

export default function createView(api: ToolboxAPI) {
  return defineComponent({
    setup() {
      // 用 api.dataSource、api.task、api.notification 等
      // 返回 render 函数或模板
    },
  });
}
```

### 7.5 插件依赖隔离

| 类别 | 处理 |
|---|---|
| `api` 模块的类 | 框架提供，插件以 `compileOnly` 方式依赖（不打包进 jar） |
| 框架内部类 | 插件**不应该**依赖；插件 ClassLoader 看不到 |
| Spring Boot / Spring | 插件不应该依赖；通过 `PluginContext` 间接使用框架能力 |
| 第三方库（如 Apache Commons） | 插件可以打进自己的 `lib/`，独立 ClassLoader 隔离 |

## 12. 跨平台与打包

### 12.1 开发与部署的环境差异

- **开发**：Mac（macOS）+ JDK 21（Mac 版） + IDE 直接跑 Spring Boot
- **部署**：Windows 服务器 + 自带的 Windows JRE 21
- **测试**：Mac 上的 Parallels Windows 虚拟机

### 12.2 跨平台代码规范

| 项 | 规范 |
|---|---|
| 文件路径 | 用 `java.nio.file.Path` API，永远不手拼 `\\` 或 `/` |
| 换行符 | 用 `System.lineSeparator()` 或 NIO `Files.write` |
| 文件编码 | 所有 IO 显式指定 `StandardCharsets.UTF_8` |
| 启动脚本 | Windows 提供 `.bat`，Mac 提供 `.sh` |

### 12.3 Gradle 打包任务

```kotlin
// app/build.gradle.kts
tasks.register("packageWindows") {
  doLast {
    // 1. 构建前端 (npm run build)
    // 2. 把前端产物复制到 framework/src/main/resources/static/
    // 3. 构建 Spring Boot fat jar
    // 4. 下载 Windows JRE 21 (从 Adoptium，缓存在 build/jre-cache/)
    // 5. 复制示例插件 jar 到 plugins/
    // 6. 复制启动脚本、WinSW 配置
    // 7. 打包成 dist/ToolBox-${version}-windows.zip
  }
}
```

### 12.4 部署目录（Windows 绿色版）

```
ToolBox/
├── bin/
│   ├── start.bat              ← 前台启动（命令行窗口）
│   ├── stop.bat               ← 停止
│   ├── install-service.bat    ← 注册 Windows 服务
│   ├── uninstall-service.bat  ← 卸载 Windows 服务
│   ├── toolbox.vmoptions      ← JVM 参数 (-Xms / -Xmx 等)
│   └── winsw.exe              ← WinSW 主程序
│   └── winsw.xml              ← 服务配置
├── jre/                       ← Windows JRE 21 (~80MB)
├── lib/
│   └── toolbox-app.jar        ← Spring Boot fat jar
├── plugins/                   ← 用户可丢插件 jar 进来
│   ├── sql-runner-1.0.0.jar
```

### 12.5 升级方式

- **整体替换**：解压新版本到旁边目录，把旧版本的 `config/`、`plugins/`、`plugins-data/`、`data/`、`logs/` 复制过来即可
- **插件单独升级**：替换 `plugins/` 里对应的 jar，重启服务

- 每个开发者本地都能跑（前提：装了 Docker Desktop）
- CI 上也能跑

## 附录 A：术语表

| 术语 | 含义 |
|---|---|
| 框架 | 本项目的主程序，提供插件加载、数据源管理、任务调度等基础能力 |
| 插件 | 一个独立的工具实现，由后端 jar + 前端 JS 组成 |
| 工具 | 用户视角的概念，等同于一个插件提供的入口（一个插件可能提供多个工具） |
| Tab | 主界面右侧工作区中的一个标签页，对应一个工具实例 |
| PluginContext | 框架在插件启动时注入的上下文对象，是插件访问框架能力的唯一入口 |
| 通用组件 | 框架提供的可复用前端组件（数据源选择器、SQL 面板等） |
| 数据源 | 一个数据库连接配置（MySQL 或 OceanBase MySQL 模式） |
| 后台任务 | 长时间运行的工作单元，异步执行，支持进度报告和取消 |
| Artifact | 任务产出的文件（如导出的 Excel） |
