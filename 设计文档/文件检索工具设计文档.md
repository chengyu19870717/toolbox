## 1. 功能概述

文件检索工具（File Search）是 ToolBox 的一个插件，用于在 Windows 环境下对指定目录下的办公文档（Word、Excel、PPT 及 WPS 对应格式）建立本地索引，并提供模糊搜索、文件路径查看与文件管理器定位功能。

### 1.1 核心能力

| 能力 | 说明 |
|---|---|
| 目录管理 | 添加/删除检索目录，支持自定义显示名称 |
| 文件扫描 | 递归遍历目录，过滤指定文档类型，建立本地 SQLite 索引 |
| 模糊搜索 | 按文件名关键字模糊匹配，支持类型过滤，分页展示 |
| 文件定位 | 在系统文件管理器（Windows 资源管理器）中打开并选中文件 |
| 索引更新 | 点击刷新按钮触发异步扫描，支持取消，进度实时推送 |

### 1.2 支持的文件格式

| 类别 | 扩展名 | 说明 |
|---|---|---|
| Word | `.doc` `.docx` | Microsoft Word |
| WPS 文字 | `.wps` | WPS Office 文字 |
| Excel | `.xls` `.xlsx` | Microsoft Excel |
| WPS 表格 | `.et` | WPS Office 表格 |
| PPT | `.ppt` `.pptx` | Microsoft PowerPoint |
| WPS 演示 | `.dps` | WPS Office 演示 |

## 2. 系统架构

### 2.1 整体位置

```
ToolBox 框架 (Spring Boot + PF4J)
    └── plugin-file-search.jar   ← 本插件
          ├── FileSearchPlugin       PF4J Plugin 主类
          ├── FileSearchExtension    ToolExtension 扩展点
          ├── FileIndexManager       数据访问层（SQLite）
          └── frontend/index.js     Vue 3 前端视图
```

### 2.2 分层架构

```
前端 Vue 组件（FileSearchView.vue）
    │ callSync(action, params) / task.submit()
    ▼
FileSearchExtension（入口层）
    │ 参数校验 → 分发
    ├── SyncHandler：listDirs / addDir / removeDir / search / stats / openFile
    └── TaskHandler：file-search-scan（异步扫描）
         │
         ▼
    FileIndexManager（业务+数据层）
         │ JDBC / SQLite
         ▼
    file_search.db（本地 SQLite 数据库）
```

> 遵循 CLAUDE.md 规范：SyncHandler 仅处理 <1s 的请求，扫描目录走 TaskHandler 异步执行。

## 3. 数据模型

### 3.1 数据库位置

```
{PluginContext.getDataDir()}/file_search.db
```

开发模式下位于 `framework/data/plugins/file-search/file_search.db`。

### 3.2 表结构

#### scan_dirs — 检索目录表

| 字段 | 类型 | 说明 |
|---|---|---|
| id | TEXT PK | UUID |
| path | TEXT UNIQUE | 目录完整路径 |
| name | TEXT | 显示名称（用户自定义） |
| last_scan | TEXT | 最近一次扫描时间（yyyy-MM-dd HH:mm:ss） |

#### file_index — 文件索引表

| 字段 | 类型 | 说明 |
|---|---|---|
| id | TEXT PK | UUID |
| dir_id | TEXT | 关联 scan_dirs.id |
| name | TEXT | 文件名（原始大小写） |
| ext | TEXT | 扩展名（小写，如 `.docx`） |
| full_path | TEXT | 文件绝对路径 |
| file_size | INTEGER | 文件大小（字节） |
| modified | TEXT | 文件最后修改时间（ISO 8601） |
| indexed | TEXT | 本次索引时间（yyyy-MM-dd HH:mm:ss） |

**索引：**
- `idx_file_name`：`file_index(name)` — 加速模糊查询
- `idx_file_dir`：`file_index(dir_id)` — 加速按目录删除

## 4. 后端接口

### 4.1 SyncHandler Actions

所有请求通过 `POST /api/plugins/file-search/sync` 发起。

#### `listDirs`
- **入参**：无
- **返回**：`{ dirs: Dir[] }`

#### `addDir`
- **入参**：`{ path: string, name?: string }`
- **说明**：若路径不存在返回 `ok: false` 和 `warn` 提示，不抛异常
- **返回**：`{ ok: boolean, warn: string }`

#### `removeDir`
- **入参**：`{ id: string }`
- **说明**：级联删除该目录下所有文件索引
- **返回**：`{ ok: true }`

#### `search`
- **入参**：`{ keyword: string, ext?: string, page: int, pageSize: int }`
- **说明**：keyword 为空时返回全部文件；ext 为空时不过滤类型
- **返回**：`{ total: int, list: FileItem[] }`

#### `stats`
- **入参**：无
- **返回**：`{ totalDirs: int, totalFiles: int }`

#### `openFile`
- **入参**：`{ path: string }`
- **说明**：调用系统文件管理器定位文件。Windows 用 `ProcessBuilder` 调用 `explorer.exe /select,<path>`，Mac 用 `open -R`，Linux 打开父目录
- **返回**：`{ ok: true }`

### 4.2 TaskHandler — `file-search-scan`

通过 `POST /api/tasks/submit` 提交。

- **type**：`file-search-scan`
- **payload**：`{ id: string }` — 目录 ID
- **进度上报**：
  - 5% — 开始收集文件列表
  - 30% — 文件列表收集完成，开始写入索引
  - 30%–95% — 每 200 条文件汇报一次进度
  - 100% — 完成
- **取消支持**：✅ 文件遍历中和批量写入中均检查 `ctx.isCancelled()`

### 5.2 组件状态

| 状态变量 | 类型 | 用途 |
|---|---|---|
| `dirs` | `Dir[]` | 检索目录列表 |
| `stats` | `{totalDirs, totalFiles}` | 底部统计 |
| `scanningId` | `string` | 当前正在扫描的目录 ID |
| `scanMsg` | `string` | 扫描进度提示文本 |
| `addWarn` | `string` | 添加目录时的警告（路径不存在） |

| `initialized` | `boolean` | 控制空状态文本（避免初始加载时闪烁） |

### 5.3 交互流程

**首次使用：**
1. 打开工具 → 自动调用 `listDirs` 和 `stats`，并触发初始搜索展示全量文件
2. 目录列表为空 → 显示引导文案
3. 点击 `+` → 输入目录路径 → `addDir` → 若路径不存在显示 Warning Alert
4. 点击 `↻` → 提交异步扫描任务 → 轮询进度 → 完成后刷新搜索

**日常搜索：**
1. 输入关键字（300ms 防抖）→ 自动触发搜索
2. 选择类型过滤 → 立即重搜（重置到第一页）
3. 点击文件行的定位按钮 → 调用 `openFile` → 资源管理器定位

## 6. 安全设计

### 6.1 命令注入防护

`openFile` 使用 `ProcessBuilder(List<String>)` 传递参数数组，OS 直接接收，路径不经 shell 解析，规避 shell injection 风险。

### 6.2 SQL 注入防护

所有 SQL 使用 `PreparedStatement` 参数化查询。LIKE 模糊查询对 `%` 和 `_` 进行转义，配合 `ESCAPE '\'` 子句。

## 7. 性能考虑

| 场景 | 设计决策 |
|---|---|
| 大目录扫描 | 走 TaskHandler 异步，不阻塞 HTTP 线程 |
| 批量写入 | `executeBatch()` + 手动事务，SQLite 每 200 条报告进度 |
| 模糊搜索 | `idx_file_name` 索引加速，分页避免一次返回大量数据 |
| 前端防抖 | 搜索框 300ms 防抖，避免每键触发请求 |
| 进度上报 | 每 200 文件汇报一次（而非每条），避免 SSE 刷屏 |

## 10. 已知限制

| 限制 | 说明 |
|---|---|
| 不支持实时监控 | 文件变更需手动点击刷新 |
| 不支持内容搜索 | 仅按文件名检索，不解析文档内容 |
| UNC 路径未验证 | 网络共享路径（\\\\server\\share）未经测试 |
| 文件定位平台差异 | Windows 资源管理器选中文件；Linux 仅打开父目录 |
| 共享索引 | 多用户共享服务端同一份文件索引 |
