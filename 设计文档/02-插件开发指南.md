# ToolBox 插件开发指南

> 版本：v1.0
> 文档定位：手把手教你从零开发一个 ToolBox 插件，包括项目结构、API 用法、调试方法、打包发布、最佳实践。
> 前置阅读：`01-架构总览.md`

---

## 1. 插件概念回顾

一个**插件 (Plugin)** 是一个独立的工具实现，包含：
- **后端代码**：Java 代码，编译后打包成 jar 内的 `classes/` 目录
- **前端代码**：Vue 组件，构建后打包成 jar 内的 `frontend/` 目录
- **元信息**：`plugin.properties` 描述插件名、版本、入口类等

最终所有内容打成**一个 jar 文件**，丢到框架的 `plugins/` 目录，重启框架后即可使用。

一个插件可以提供 1 个或多个**工具 (Tool)**，每个工具在主界面左侧工具树上显示为一个入口。

## 2. 开发前准备

### 2.1 环境

| 工具 | 版本 |
|---|---|
| JDK | 21 |
| Gradle | 8.x（项目自带 wrapper） |
| Node.js | 20+（前端构建用） |
| pnpm 或 npm | 任选 |
| IDE | IntelliJ IDEA 推荐 |

### 2.2 拿到框架源码 / 依赖

- **方式 A：在主项目里开发插件**——把插件作为主项目的子模块，最方便调试
- **方式 B：独立项目开发插件**——主项目发布 `api` 和 `frontend-sdk` 到内部仓库（或本地 Maven 仓库），插件项目作为独立工程引用

新手建议从**方式 A** 开始，本指南也按方式 A 演示。

## 3. 创建一个新插件

### 3.1 用脚手架快速创建

主项目根目录提供一个脚手架命令：

```bash
# 在主项目根目录执行
./gradlew :tools:newPlugin -PpluginId=hello-world -PpluginName="HelloWorld 演示"
```

这会自动生成：

```
plugin-hello-world/
├── build.gradle.kts
├── backend/
│   └── src/main/
│       ├── java/com/toolbox/plugin/helloworld/
│       │   ├── HelloWorldPlugin.java
│       │   └── HelloWorldExtension.java
│       └── resources/
│           └── plugin.properties
└── frontend/
    ├── package.json
    ├── vite.config.ts
    └── src/
        ├── index.ts
        ├── HelloWorldView.vue
        └── api.d.ts
```

并在 `settings.gradle.kts` 里自动添加这个模块。

### 3.2 不用脚手架手动创建

参考下文章节 5 的"插件项目结构"，照着模板抄一份。

## 4. 插件元信息：plugin.properties

```properties
# PF4J 标准字段
plugin.id=hello-world                 # 唯一 ID，全局不能重复
plugin.class=com.toolbox.plugin.helloworld.HelloWorldPlugin
plugin.version=1.0.0                  # 语义化版本
plugin.provider=ToolBox Team
plugin.description=HelloWorld 演示插件
plugin.dependencies=                  # 依赖其他插件（用逗号分隔），可空

# ToolBox 自定义字段
toolbox.category=演示                  # 工具树分组（不写归到"其他"）
toolbox.icon=mdi-hand-wave            # Material Design Icon 名
toolbox.frontendEntry=frontend/index.js  # 前端入口路径（jar 内相对路径）
toolbox.minFrameworkVersion=1.0.0     # 要求的最低框架版本
```

**字段约束**：

| 字段 | 必填 | 说明 |
|---|---|---|
| plugin.id | ✓ | 小写字母+数字+短横线，例如 `sql-runner` |
| plugin.class | ✓ | 完整类名 |
| plugin.version | ✓ | 语义化版本 |
| plugin.provider | | 作者/团队名 |
| plugin.description | | 简短描述（一句话） |
| plugin.dependencies | | `pluginA, pluginB@>=1.2.0` |
| toolbox.category | | 不写默认归"其他" |
| toolbox.icon | | 不写用默认图标 |
| toolbox.frontendEntry | ✓ | 前端入口 |
| toolbox.minFrameworkVersion | | 不满足时拒绝加载 |

### 5.2 build.gradle.kts 模板

```kotlin
plugins {
    java
    id("com.toolbox.plugin")  // 主项目自定义 plugin，封装打包逻辑
}

group = "com.toolbox.plugin"
version = "1.0.0"

dependencies {
    // 框架 API（不打进插件 jar，由框架提供）
    compileOnly(project(":api"))
    compileOnly("org.pf4j:pf4j:3.10.0")
    compileOnly("org.slf4j:slf4j-api:2.0.9")

    // 插件私有依赖（会打进 lib/）
    implementation("com.example:my-special-lib:1.0.0")

    // 测试
    testImplementation("org.junit.jupiter:junit-jupiter:5.10.0")
    testImplementation("org.mockito:mockito-core:5.5.0")
    testImplementation(project(":api"))
}

// 自定义任务：构建前端并复制到 jar 里
tasks.named<Jar>("jar") {
    dependsOn("buildFrontend")

    // 把前端构建产物作为 jar 内的 frontend/ 目录
    from("frontend/dist") {
        into("frontend")
    }

    // 把私有依赖打进 lib/
    from(configurations.runtimeClasspath) {
        into("lib")
    }
}

tasks.register<Exec>("buildFrontend") {
    workingDir = file("frontend")
    commandLine("npm", "run", "build")
}
```

## 6. 后端开发

### 6.1 插件入口：Plugin 类

```java
package com.toolbox.plugin.helloworld;

import com.toolbox.api.plugin.PluginContext;
import com.toolbox.api.plugin.ToolBoxPlugin;
import org.pf4j.Plugin;
import org.pf4j.PluginWrapper;
import org.slf4j.Logger;

public class HelloWorldPlugin extends Plugin implements ToolBoxPlugin {

    private PluginContext context;
    private Logger log;

    public HelloWorldPlugin(PluginWrapper wrapper) {
        super(wrapper);
    }

    @Override
    public void init(PluginContext context) {
        this.context = context;
        this.log = context.getLogger();
        log.info("HelloWorldPlugin initializing...");
    }

    @Override
    public void start() {
        log.info("HelloWorldPlugin started");
    }

    @Override
    public void stop() {
        log.info("HelloWorldPlugin stopping...");
        // 释放资源（关闭线程池、断开外部连接等）
    }

    public PluginContext getContext() {
        return context;
    }
}
```

**生命周期顺序**：
```
构造 (Constructor) → init(context) → start() → ... 运行 ... → stop()
```

`init()` 是框架在加载插件时调用，比 PF4J 标准的 `start()` 更早，用于注入 `PluginContext`。

### 6.2 工具扩展点：ToolExtension 实现

每个工具实现 `ToolExtension` 接口：

```java
package com.toolbox.plugin.helloworld;

import com.toolbox.api.plugin.PluginContext;
import com.toolbox.api.plugin.ToolExtension;
import com.toolbox.api.plugin.handler.SyncHandler;
import com.toolbox.api.plugin.handler.TaskHandler;
import org.pf4j.Extension;

@Extension
public class HelloWorldExtension implements ToolExtension {

    private PluginContext context;

    @Override
    public void init(PluginContext context) {
        this.context = context;
    }

    @Override
    public String getId() { return "hello-world-main"; }

    @Override
    public String getName() { return "HelloWorld 演示"; }

    @Override
    public String getCategory() { return "演示"; }

    @Override
    public String getIcon() { return "mdi-hand-wave"; }

    @Override
    public SyncHandler getSyncHandler() {
        return new HelloSyncHandler(context);
    }

    @Override
    public TaskHandler getTaskHandler() {
        return new HelloTaskHandler(context);
    }
}
```

**SyncHandler vs TaskHandler**：
- `SyncHandler` 处理**同步**请求（用户点击 → 后端立即返回结果），比如执行一条 SQL
- `TaskHandler` 处理**异步**长任务（提交后台 → 进度推送 → 完成时返回产物），比如导入大 Excel
- 二者都可以为 null，按工具类型选择

### 6.3 SyncHandler

```java
public class HelloSyncHandler implements SyncHandler {

    private final PluginContext context;

    public HelloSyncHandler(PluginContext context) {
        this.context = context;
    }

    @Override
    public Object handle(String action, Map<String, Object> params) {
        switch (action) {
            case "greet":
                String name = (String) params.get("name");
                return Map.of("message", "Hello, " + name + "!");
            case "queryUsers":
                return queryUsers((String) params.get("dataSourceId"));
            default:
                throw new IllegalArgumentException("Unknown action: " + action);
        }
    }

    private List<Map<String, Object>> queryUsers(String dataSourceId) {
        QueryRequest req = QueryRequest.builder()
            .dataSourceId(dataSourceId)
            .sql("SELECT id, name FROM users LIMIT 100")
            .build();
        QueryResult result = context.getDataSourceManager().query(req);
        return result.toMapList();
    }
}
```

前端调用：
```typescript
const result = await api.plugin.callSync('greet', { name: 'Alice' });
// result.message === "Hello, Alice!"
```

### 6.4 TaskHandler

```java
public class HelloTaskHandler implements TaskHandler {

    private final PluginContext context;

    public HelloTaskHandler(PluginContext context) {
        this.context = context;
    }

    @Override
    public TaskResult execute(TaskExecutionContext ctx) throws Exception {
        Map<String, Object> payload = ctx.getPayload();
        ProgressReporter reporter = ctx.getProgressReporter();

        int total = 100;
        for (int i = 0; i < total; i++) {
            // 关键：每次迭代都要响应中断
            if (Thread.currentThread().isInterrupted()) {
                throw new InterruptedException("Task cancelled");
            }

            // 业务逻辑
            doStep(i);

            // 报告进度
            reporter.report((i + 1) * 100 / total, "处理第 " + (i + 1) + " 项");
        }

        // 生成产物
        Path outputFile = context.getDataDir().resolve("output.txt");
        Files.writeString(outputFile, "Hello!", StandardCharsets.UTF_8);

        return TaskResult.builder()
            .addArtifact(TaskArtifact.builder()
                .name("output.txt")
                .path(outputFile.toString())
                .contentType("text/plain")
                .build())
            .build();
    }

    private void doStep(int i) {
        // 业务逻辑
    }
}
```

**响应中断的几种正确写法**：

```java
// 写法 1: 在循环里检查
while (hasMore()) {
    if (Thread.currentThread().isInterrupted()) {
        throw new InterruptedException();
    }
    process();
}

// 写法 2: 调用本身就会响应中断的方法（推荐）
Thread.sleep(1000);  // 抛 InterruptedException
queue.poll(1, TimeUnit.SECONDS);  // 抛 InterruptedException

// 写法 3: 数据库查询
try (Statement stmt = conn.createStatement()) {
    // 框架会在取消时调用 stmt.cancel()
    stmt.executeQuery(sql);
}
```

**⚠ 重要**：`InterruptedException` 不要吞掉，要么往外抛，要么 `Thread.currentThread().interrupt()` 重新设置中断标志。

### 6.5 调用框架能力

通过 `PluginContext` 拿到框架提供的所有能力。详细 API 见《04-后端 API 参考》。常用举例：

```java
// 数据源
DataSourceManager dsm = context.getDataSourceManager();
QueryResult result = dsm.query(QueryRequest.builder()
    .dataSourceId(dsId)
    .sql("SELECT * FROM ?")
    .params(List.of(123))
    .timeoutSeconds(30)
    .maxRows(10000)
    .build());

// 任务管理（一般不直接用，TaskHandler 由框架自动调用）
TaskManager tm = context.getTaskManager();

// 通知
NotificationService notifier = context.getNotificationService();
notifier.info(context.getCurrentUser(), "处理完成");

// 私有数据目录
Path dataDir = context.getDataDir();  // plugins-data/hello-world/
Path config = dataDir.resolve("config.json");

// 日志
Logger log = context.getLogger();
log.info("Hello, {}!", name);
```

### 6.6 异常处理

抛出框架定义的标准异常，框架会自动展示给用户：

```java
import com.toolbox.api.exception.*;

if (dataSourceId == null) {
    throw new ValidationException("请先选择数据源");
}

try {
    dsm.query(req);
} catch (DataSourceException e) {
    // 框架的 DataSourceException 会自动展示，无需处理
    throw e;
}

throw new ToolBoxException("自定义业务错误");
```

不要 `catch Exception` 然后吞掉——让异常往上抛，框架会统一处理。

## 7. 前端开发

### 7.1 入口：index.ts

```typescript
import type { ToolboxAPI, ToolView } from '@toolbox/frontend-sdk';
import { defineComponent, h } from 'vue';
import HelloWorldView from './HelloWorldView.vue';

/**
 * 插件入口函数。框架在创建 Tab 时调用。
 * 返回一个 Vue 组件构造器，框架会用它实例化 Tab 内容。
 */
export default function createView(api: ToolboxAPI, toolId: string): ToolView {
  return defineComponent({
    name: 'HelloWorldView',
    setup() {
      // 把 api 传给子组件
      return () => h(HelloWorldView, { api });
    },
  });
}
```

### 7.2 主视图组件

```vue
<!-- HelloWorldView.vue -->
<template>
  <div class="hello-world">
    <DataSourceSelector v-model="dataSourceId" @change="onDsChange" />

    <el-input v-model="name" placeholder="请输入姓名" style="width: 300px" />
    <el-button type="primary" @click="greet">问好</el-button>

    <div v-if="message" class="message">{{ message }}</div>

    <el-button @click="runLongTask">运行长任务</el-button>
    <ProgressPanel v-if="taskId" :task-id="taskId" @cancel="onCancel" />
  </div>
</template>

<script setup lang="ts">
import { ref } from 'vue';
import type { ToolboxAPI } from '@toolbox/frontend-sdk';
import { DataSourceSelector, ProgressPanel } from '@toolbox/frontend-sdk';

const props = defineProps<{ api: ToolboxAPI }>();

const dataSourceId = ref<string | null>(null);
const name = ref('');
const message = ref('');
const taskId = ref<string | null>(null);

async function greet() {
  try {
    const result = await props.api.plugin.callSync('greet', { name: name.value });
    message.value = result.message;
  } catch (e) {
    // 框架已经统一弹出错误对话框，这里不用再处理
  }
}

async function runLongTask() {
  const { taskId: id } = await props.api.task.submit({
    name: 'HelloWorld 长任务',
    payload: { items: 100 },
  });
  taskId.value = id;
}

async function onCancel() {
  if (taskId.value) {
    await props.api.task.cancel(taskId.value);
  }
}

function onDsChange(id: string) {
  // 改 Tab 标题，让用户能区分
  props.api.tab.setTitle(`HelloWorld - ${id}`);
}
</script>

<style scoped>
.hello-world {
  padding: 16px;
}
.message {
  margin-top: 16px;
  color: var(--el-color-success);
}
</style>
```

### 7.3 ToolboxAPI 概览

`ToolboxAPI` 是框架注入给插件前端的全局对象，包含：

```typescript
interface ToolboxAPI {
  // 插件后端调用
  plugin: {
    callSync(action: string, params: Record<string, any>): Promise<any>;
  };

  // 数据源
  dataSource: {
    list(): Promise<DataSource[]>;
    connect(id: string): Promise<void>;
    release(id: string): Promise<void>;
    query(req: QueryRequest): Promise<QueryResult>;
  };

  // 任务
  task: {
    submit(req: TaskSubmitRequest): Promise<{ taskId: string }>;
    cancel(taskId: string): Promise<void>;
    listMine(): Promise<Task[]>;
    subscribe(taskId: string, handlers: ProgressHandlers): () => void;
  };

  // 文件
  file: {
    upload(file: File, onProgress?: (percent: number) => void): Promise<UploadedFile>;
    downloadUrl(taskId: string, artifactName: string): string;
  };

  // 通知
  notification: {
    success(msg: string): void;
    info(msg: string): void;
    warning(msg: string): void;
    error(msg: string): void;
  };

  // Tab 控制
  tab: {
    setTitle(title: string): void;
    close(): void;
  };

  // 当前用户
  user: {
    username: string;
    role: 'USER' | 'ADMIN';
  };
}
```

完整 API 详见《03-前端组件 API 参考》。

### 7.5 不要做的事

| 禁忌 | 原因 |
|---|---|
| 直接 fetch 后端 `/api/**` | 应该用 `api.plugin.callSync` 等封装方法（自动带 token） |
| import 非 SDK 的 vue 包 | 主程序已经引入了 vue / element-plus，重复打包会出问题 |
| 修改 `document.title` 等全局 DOM | 影响其他 Tab，应该用 `api.tab.setTitle` |
| 操作 `localStorage` 存敏感数据 | 多用户共用浏览器场景下不安全 |
| 注册全局快捷键 | 第一版不支持，后续可能开放 |
| 长时间占用主线程 | 长任务必须用 `TaskHandler`，前端只 UI，业务在后端 |

## 8. 调试

### 8.1 后端调试

**方式 A：把插件作为主项目子模块（推荐）**

```bash
# 主项目根目录
./gradlew :framework:bootRun
```

主项目启动时会扫描 `plugins/` 目录，但开发时插件代码还没打成 jar——解决方法：在主项目 `framework/build.gradle.kts` 里加一个 dev profile：

```kotlin
// 开发模式：直接把插件子模块加到 classpath，不走 PF4J
val devPlugins = if (project.hasProperty("dev")) {
    configurations.create("devPluginRuntime")
} else null

dependencies {
    if (project.hasProperty("dev")) {
        "devPluginRuntime"(project(":plugin-hello-world"))
    }
}
```

启动时加 `-Pdev` 参数即可热调试：

```bash
./gradlew :framework:bootRun -Pdev
```

IDE 里直接对插件代码下断点。

**方式 B：先打包再放进 plugins/**

```bash
./gradlew :plugin-hello-world:jar
cp plugin-hello-world/build/libs/plugin-hello-world-1.0.0.jar runtime/plugins/
./gradlew :framework:bootRun
```

每次改代码都要重新打包重启，慢但接近真实环境。

### 8.2 前端调试

```bash
cd plugin-hello-world/frontend
npm run dev
```

Vite 启动 dev server（默认 5173 端口），但插件前端是要嵌入主程序的，单独跑 dev server 没意义。**正确做法**：

```bash
# 主项目 frontend 启动 dev 模式
cd frontend
npm run dev   # 默认 3000 端口

# 配置代理：把 /plugins/hello-world/* 代理到插件源码
# vite.config.ts:
server: {
  proxy: {
    '/plugins/hello-world': {
      target: 'http://localhost:5173',
      rewrite: (path) => path.replace('/plugins/hello-world', ''),
    }
  }
}
```

### 8.4 常见问题

**Q: 启动报 `ClassNotFoundException`？**
A: 检查 plugin.properties 里的 `plugin.class` 是不是写错了完整类名。

**Q: 前端报 `Cannot find module '@toolbox/frontend-sdk'`？**
A: 在主项目根目录跑 `pnpm install`（或 npm），让 workspace 链接生效。

**Q: 插件加载了但工具树看不到？**
A: 检查 `ToolExtension` 是不是加了 `@Extension` 注解；检查 `META-INF/extensions.idx` 是不是生成了（PF4J 要求）。

**Q: 后台任务永远不响应取消？**
A: 检查 `TaskHandler.execute()` 里有没有定期检查 `Thread.currentThread().isInterrupted()`，长循环必须加。

**Q: 前端组件改了样式但没生效？**
A: 浏览器强制刷新（Cmd+Shift+R），动态加载的 CSS 缓存比较顽固。

## 11. 最佳实践

### 11.1 命名

- plugin.id 用小写 + 短横线：`sql-runner`、`order-flow-viewer`
- 包名用 plugin.id 替换短横线为下划线或合并：`com.toolbox.plugin.sqlrunner`
- 工具 ID 加前缀避免冲突：`hello-world-main`、`hello-world-debug`

### 11.2 错误信息

- 给用户看的错误：清晰、可操作（"请先在数据源管理添加 MySQL 数据源"）
- 给开发者看的：日志里 + 堆栈
- 不要把内部技术细节暴露给用户（"NullPointerException at line 47"）

### 11.3 长任务

- 业务一定能在 1 秒内返回 → SyncHandler
- 可能超过 1 秒 → TaskHandler（哪怕实际很快，也比 UI 卡死好）
- 任务里**必须**响应中断
- 进度上报频率：建议 1 秒最多 1-2 次（太多会刷屏 SSE）

### 11.4 资源管理

- 数据库连接：用 try-with-resources，绝不手动 close
- 文件句柄：try-with-resources
- 临时文件：放 `context.getDataDir()` 或 framework 的 temp 目录
- 不要在插件里创建线程池——用 `TaskHandler`

### 11.5 配置

- 插件级配置（所有用户共享）：`context.getDataDir()/config.json`
- 用户级配置（每用户独立）：`context.getDataDir()/users/{username}/config.json`
- 不要直接读 framework 的 application.yml
- 频繁读写的配置加内存缓存

### 11.6 前端组件复用

- 优先用 `@toolbox/frontend-sdk` 的通用组件
- 用 Element Plus（已经引入）
- 不要引入大量额外依赖（每个插件 jar 体积会膨胀）

### 11.7 日志

- 关键路径打 INFO（"开始处理 1000 条数据"）
- 详细信息打 DEBUG（"处理第 1 条: id=123"）
- 错误打 ERROR + 堆栈
- 不要在循环里打 INFO（日志爆炸）

### 11.8 测试

- 业务核心逻辑必须有单元测试
- 数据库相关用 Testcontainers，不连真实库
- 前端测试关注**行为**（点了按钮做了什么），不测样式

### 11.9 版本管理

- 语义化版本：MAJOR.MINOR.PATCH
- 不兼容修改：MAJOR+1
- 加功能：MINOR+1
- 修 bug：PATCH+1
- `plugin.properties` 里的 version 一定要跟 build.gradle.kts 同步

### 11.10 文档

- 每个插件项目根目录写一个 README.md：
  - 插件做什么
  - 怎么用（截图）
  - 配置项
  - 已知限制
  - changelog

## 13. FAQ

**Q: 一个插件能提供多个工具吗？**
A: 能。在同一个插件里写多个 `ToolExtension` 实现类，每个都加 `@Extension` 注解。它们会在工具树上显示为多个独立入口。

**Q: 插件之间能互相调用吗？**
A: 第一版不支持。如果有共享代码需求，抽到独立的 jar 库，每个插件单独依赖。

**Q: 插件能监听其他插件的事件吗？**
A: 不行。第一版不做插件间通信。

**Q: 插件能改主界面（菜单栏、工具树）吗？**
A: 不能。插件只能在自己的 Tab 内渲染 UI。注册菜单项等能力可能在未来版本开放。

**Q: 插件能调用外部 HTTP 接口吗？**
A: 可以。但建议在后端用（在 TaskHandler / SyncHandler 里），别在浏览器跨域调外部接口。

**Q: 插件能读 framework 的配置吗？**
A: 不行。framework 配置是私有的，插件通过 PluginContext 提供的方法访问需要的能力。

**Q: 我的插件需要保存大量数据，用什么？**
A: 文件用 `context.getDataDir()`。结构化数据可以用 SQLite（自己引一份 sqlite-jdbc），但建议先用 JSON 文件，简单到极致后再考虑数据库。

**Q: 插件能用最新的 Java 21 特性吗？**
A: 可以，框架要求 JDK 21，所以语法、API 全部可用。

**Q: 我的插件依赖 PostgreSQL，能用吗？**
A: 不能直接通过框架的 DataSourceManager（只支持 MySQL/OB）。可以在插件里自己连——建表、查询都自己写，把 PostgreSQL 驱动打进 lib/。
