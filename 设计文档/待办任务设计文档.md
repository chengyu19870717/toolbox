## 一、项目概述

### 1.1 产品定位

Toolbox 是面向银行业务场景的轻量级内网工具箱，提供常用效率工具的统一入口。**纯内网可用，零外部依赖。**

### 1.2 功能模块

| 模块 | 说明 | 状态 |
|------|------|------|
| **流程解析工具** | 解析审批流程XML，可视化展示+Excel导出 | V1.0 已设计 |
| **待办任务管理** | 按天管理待办，支持任务拆分、自动顺延、每周复盘 | V3.0 重新设计 |

## 三、模块二：待办任务管理

### 3.1 核心诉求

| # | 诉求 | 设计关键词 |
|---|------|-----------|
| 1 | 便捷操作 | 输入即保存，最少步骤 |
| 2 | 按天新增待办 | 日期驱动，每天一个任务清单 |
| 3 | 任务支持拆分 | 父子任务，大任务拆小 |
| 4 | 设置最晚办结日期 | 截止日期，逾期标红 |
| 5 | 未办结自动顺延 | 每天自动挪到下一天 |
| 6 | 导入导出 | JSON/Excel 双格式 |
| 7 | 每周复盘 | 自动生成周报统计 |
| 8 | 内网环境 | 纯前端，零外部依赖 |
| 9 | 集成到toolbox | 作为toolbox入口之一 |

### 3.2 数据模型

#### 3.2.1 任务 Task

```typescript
interface Task {
  id: string;              // 唯一标识（UUID）
  title: string;           // 任务标题（必填）
  description?: string;    // 任务描述（选填）
  parentId?: string;       // 父任务ID（空=顶层任务）
  
  // 状态
  status: TaskStatus;      // todo / completed
  isSplit: boolean;        // 是否已拆分（有子任务=已拆分）
  
  // 日期
  targetDate: string;      // 目标办结日期（ISO格式 YYYY-MM-DD，必填）
  // 注：初始值=创建当天的日期；顺延时更新为当天日期（打开页面当天）
  
  // 项目分类
  project?: string;        // 所属项目名称（选填，简单的字符串即可）
  
  // 干系人
  stakeholders?: Stakeholder[]; // 任务干系人列表（选填）
  
  // 时间戳
  createdAt: string;       // 创建时间
  updatedAt: string;       // 最后更新时间
  completedAt?: string;    // 完成时间
  originalTargetDate: string; // 原始目标日期（首次创建时的日期，不随顺延变化）
  rolloverCount: number;   // 顺延次数（从0开始，每顺延一次+1）
  
  // 排序
  sortIndex: number;       // 当日列表内排序权重
}

enum TaskStatus {
  TODO = 'todo',
  COMPLETED = 'completed'
}
```

#### 3.2.2 干系人 Stakeholder

```typescript
interface Stakeholder {
  id: string;              // 唯一标识
  name: string;            // 干系人姓名（必填）
  role?: string;           // 角色/身份（选填，如：需求方、开发、审批人）
  remark?: string;         // 备注信息（选填，如：联系方式、关注点、协作说明）
  addedAt: string;         // 添加时间
}
```

#### 3.2.3 项目 Project

```typescript
interface Project {
```

#### 3.2.4 周报 WeeklyReport

```typescript
interface WeeklyReport {
  weekStart: string;       // 周一日期 YYYY-MM-DD
  weekEnd: string;         // 周日日期 YYYY-MM-DD
  generatedAt: string;     // 生成时间
  
  // 统计
  totalCreated: number;    // 本周新建任务数
  totalCompleted: number;  // 本周完成任务数
  totalRollover: number;   // 本周顺延任务数
  totalOverdue: number;    // 本周逾期任务数
  completionRate: number;  // 完成率（%）
  
  // 明细
  dailyStats: DailyStat[]; // 每日统计
  overdueTasks: Task[];    // 逾期未完成任务
  projectStats: {name: string; created: number; completed: number}[]; // 按项目统计
}

interface DailyStat {
  date: string;            // 日期
  created: number;         // 当日新建
  completed: number;       // 当日完成
  rollover: number;        // 当日顺延到下一天
}
```

### 3.4 核心交互设计

#### 3.4.1 新建任务

**最快路径（2步完成）：**

```
第1步：点击「+ 新任务」→ 出现输入框
第2步：输入标题 → 回车保存
```

默认值：
- targetDate = 当天日期
- status = todo
- project = 无

**快速输入语法（可选）：**

| 输入示例 | 解析 |
|---------|------|
| `完成文档 📅5/15 #流程` | 标题=完成文档，截止日期=5/15，项目=流程 |
| `审核方案 📅周五` | 标题=审核方案，截止日期=本周五 |
| `更新需求 📅+3` | 标题=更新需求，截止日期=3天后 |
| `跟进反馈 #日常` | 标题=跟进反馈，项目=日常 |

**日期语法：**

| 语法 | 含义 |
|------|------|
| `📅5/15` / `📅5-15` | 具体日期 |
| `📅今天` / `📅明天` / `后天` | 相对日期 |
| `📅周五` / `📅下周一` | 星期 |
| `📅+3` / `📅-2` | N天后/N天前 |

#### 3.4.2 任务拆分

**操作流程：**

```
任务卡片 → 右键/更多按钮 → 「拆分任务」
                           ↓
              ┌──────────────────────────────┐
              │  拆分任务                      │
              │                               │
              │  原任务：完成授信流程节点梳理     │
              │                               │
              │  子任务1：[整理角色编码翻译表  ]  │
              │  子任务2：[整理机构层级编码    ]  │
              │  子任务3：[导出Excel验证      ]  │
              │                               │
              │  [+ 添加子任务]                 │
              │                               │
              │         [取消]  [确认拆分]      │
              └──────────────────────────────┘
```

**拆分规则：**

| 规则 | 说明 |
|------|------|
| 层级限制 | **仅支持两层**（父任务 → 子任务），子任务不可再拆分 |
| 拆分后父任务 | 标记 `isSplit=true`，不再独立显示在清单中 |
| 子任务继承 | 截止日期、项目、**干系人** 均继承父任务（子任务可单独修改） |
| 子任务状态 | 默认 todo |
| 完成判定 | 全部子任务完成 → 父任务自动标记完成 |
| 反向操作 | 可将父任务「取消拆分」，子任务变为独立任务 |

**干系人继承细节：**
- 拆分时，子任务复制父任务的全部干系人（深拷贝，各自独立修改互不影响）
- 用户可在子任务编辑面板中单独增删改子任务的干系人
- 父任务干系人变更不会自动同步到已拆分的子任务（避免覆盖用户已做的个性化调整）

#### 3.4.3 完成任务

| 操作 | 效果 |
|------|------|
| 点击顶层任务勾选框 | 直接标记完成，`completedAt` 记录当前时间 |
| 点击子任务勾选框 | 标记子任务完成；全部子任务完成则父任务自动完成 |
| 撤销顶层任务完成 | 恢复为待办，**`targetDate` 重置为撤销当天的日期** |
| 撤销子任务完成 | 恢复为待办，`targetDate` 重置为撤销当天的日期 |
| 撤销父任务完成（子任务部分完成） | 父任务恢复为待办，`targetDate` 重置为撤销当天的日期；未完成的子任务保持原状态不变 |

**撤销完成 targetDate 重置规则：**

| 场景 | targetDate 处理 | 理由 |
|------|----------------|------|
| 当天完成的任务撤销 | 不变（仍为今天） | 无需调整 |
| 历史任务撤销 | 重置为**撤销当天的日期** | 避免撤销后立即被顺延，给用户重新处理的机会 |

### 3.5 自动顺延机制

#### 3.5.1 顺延规则

| 规则 | 说明 |
|------|------|
| 触发条件 | 每次打开页面时检查，或每天00:00定时检查 |
| 顺延对象 | `status=todo` 且 `targetDate < 今天` 的任务 |
| 顺延动作 | `targetDate` 更新为**今天**（即用户打开页面当天的日期），任务出现在当天清单中 |
| 顺延与完成 | **已完成任务不参与顺延**，无论其 targetDate 是什么日期 |
| 顺延计数 | `rolloverCount += 1`，`originalTargetDate` 保持不变 |
| 父子联动 | 父任务顺延时，**未完成**的子任务 `targetDate` 一并更新为今天 |
| 子任务单独 | 子任务不会单独顺延，跟随父任务；已完成子任务不受影响 |
| 已完成子任务 | 父任务顺延时，已完成的子任务保持原状（不改变 `targetDate`） |

#### 3.5.2 顺延标记

顺延后的任务卡片显示原定日期：

```
┌────────────────────────────────────────────────────┐
│ ☐  整理会议纪要                   📅5/12→今天  #日常  │
│    (原定5/10，已顺延2天)                            │
└────────────────────────────────────────────────────┘
```

| 场景 | 显示 |
|------|------|
| 未顺延 | `📅5/12` |
| 顺延1次 | `📅5/12→今天` |
| 顺延2次 | `📅5/10→今天 (原定5/10，已顺延2天)` |

#### 3.5.3 顺延统计

顺延次数纳入每周复盘统计，用于识别拖延项。

#### 3.9.3 顺延检查时机

```javascript
// 每次进入待办任务页面时执行
function checkAndRollover() {
  const today = getToday(); // YYYY-MM-DD
  
  const overdueTasks = tasks.filter(t => 
    t.status === 'todo' && 
    t.targetDate < today &&
    !t.parentId // 只处理顶层任务，子任务跟随父任务
  );
  
  overdueTasks.forEach(task => {
    task.targetDate = today;
    task.rolloverCount += 1;
    task.updatedAt = now();
    
    // 顺延未完成的子任务
    if (task.isSplit) {
      const children = tasks.filter(t => t.parentId === task.id && t.status === 'todo');
      children.forEach(child => {
        child.targetDate = today;
        child.updatedAt = now();
      });
    }
  });
  
  saveTasks(tasks);
}
```
